# UX Design Specification - AI-as-Me v3.1

**Author:** Jody
**Date:** 2026-01-15

---

## Project Understanding

### Product Overview

**AI-as-Me v3.1** 是一个自我进化的 AI 代理系统，v3.1 版本聚焦于提升可用性，让"AI 养蛊"理念从技术原型转变为可用产品。

**核心价值主张：**
- 从"能进化"到"可验证的进化"
- 5 分钟内让用户看到 AI 自己生成规则
- 规则冲突零容忍，确保系统稳定
- 数据驱动的进化效果量化

### Target Users

#### Primary User 1: 新用户（探索者）
- **典型代表**：李明，28 岁软件工程师
- **目标**：快速体验"AI 养蛊"效果
- **痛点**：不知道如何触发首次进化
- **成功标准**：5-10 分钟内看到 AI 生成第一条规则

#### Primary User 2: 持续使用者（优化者）
- **典型代表**：王芳，35 岁产品经理
- **目标**：确保 AI 生成的规则有效且不冲突
- **痛点**：担心规则冲突破坏系统
- **成功标准**：能基于数据管理规则

#### Primary User 3: 开发者（贡献者）
- **典型代表**：张伟，32 岁开源贡献者
- **目标**：改进进化算法并验证效果
- **痛点**：缺少量化指标
- **成功标准**：能用数据证明改进价值

### Key Features (v3.1)

#### 1. 首次进化示例
- **用户价值**：降低体验门槛，5 分钟看到效果
- **核心交互**：一键 Demo → 实时进度 → 查看生成的规则

#### 2. 规则冲突检测
- **用户价值**：系统稳定可控，敢于启用进化
- **核心交互**：自动扫描 → 冲突警告 → 自动处理

#### 3. 增强进化统计
- **用户价值**：量化进化效果，数据驱动决策
- **核心交互**：查看统计 → 分析趋势 → 管理规则

### UX Challenges

**挑战 1：复杂概念的可视化**
- "进化闭环"是抽象概念，如何让用户直观理解？
- 解决方案：实时进度展示 + 可视化流程图

**挑战 2：数据密集的界面**
- 进化统计包含大量数据，如何不让用户overwhelmed？
- 解决方案：分层展示，从概览到详情

**挑战 3：首次体验的关键性**
- 新用户的"啊哈"时刻决定留存
- 解决方案：引导式 Demo，确保成功率 >90%

## User Flows

### Flow 1: 首次进化体验（新用户）

```
[登录] → [欢迎页面]
         ↓
    [看到"开始首次进化"CTA]
         ↓
    [点击"开始 Demo"]
         ↓
    [进化进度页面]
    - 步骤 1/5: 执行示例任务 ✓
    - 步骤 2/5: 收集经验 ✓
    - 步骤 3/5: 识别模式 ⏳
    - 步骤 4/5: 生成规则
    - 步骤 5/5: 写入 Soul
         ↓
    [完成页面]
    - 显示生成的规则
    - "查看规则文件"按钮
    - "查看进化统计"按钮
         ↓
    [引导到进化中心]
```

### Flow 2: 规则冲突检测（持续使用者）

```
[进化中心] → [规则管理]
              ↓
         [看到冲突警告徽章]
              ↓
         [点击"查看冲突"]
              ↓
         [冲突详情页面]
         - 冲突列表
         - 每个冲突显示：
           * Core Rule vs Learned Rule
           * 冲突类型
           * 建议处理方式
              ↓
         [选择处理方式]
         - 保留 Core（降级 Learned）
         - 修改 Core
         - 删除 Learned
              ↓
         [确认处理]
              ↓
         [返回规则管理]
```

### Flow 3: 进化统计分析（开发者）

```
[进化中心] → [进化统计]
              ↓
         [概览 Dashboard]
         - 总规则数
         - 应用率
         - 平均有效性
              ↓
         [选择查看详情]
              ↓
         [详细统计页面]
         - 规则应用热力图
         - 有效性趋势图
         - 进化时间线
              ↓
         [点击单条规则]
              ↓
         [规则详情页面]
         - 应用次数
         - 有效性评分
         - 应用历史
         - 相关经验
```

## Visual Design

### Design System (复用 v3.0)

**颜色方案：**
- Primary: #4F46E5 (Indigo)
- Success: #10B981 (Green)
- Warning: #F59E0B (Amber)
- Error: #EF4444 (Red)
- Neutral: #6B7280 (Gray)

**Typography：**
- Heading: Inter, Bold
- Body: Inter, Regular
- Code: JetBrains Mono

**Spacing：**
- 基础单位：8px
- 组件间距：16px, 24px, 32px

### v3.1 新增视觉元素

**进化进度指示器：**
- 使用动画圆点表示进行中
- 绿色勾表示完成
- 灰色圆圈表示未开始

**规则冲突徽章：**
- 红色警告图标
- 数字徽章显示冲突数量

**统计图表：**
- 热力图：渐变色表示应用频率
- 趋势图：折线图表示有效性变化
- 时间线：垂直时间轴展示进化历史

## Interaction Patterns

### 首次进化引导

**模式：引导式流程**
- 单一路径，无分支
- 每步显示进度
- 不可跳过，确保完整体验
- 完成后给予正向反馈

### 规则冲突处理

**模式：决策辅助**
- 清晰展示冲突双方
- 提供建议处理方式
- 支持一键自动处理
- 支持手动精细控制

### 进化统计浏览

**模式：渐进式披露**
- 概览 → 详情 → 单项
- 支持筛选和排序
- 支持导出数据

## Responsive Design

**桌面优先：**
- 主要使用场景是桌面浏览器
- 最小宽度：1280px

**移动适配：**
- 响应式布局
- 简化的移动界面
- 关键功能可用

## Accessibility

**WCAG 2.1 AA 标准：**
- 颜色对比度 ≥ 4.5:1
- 键盘导航支持
- 屏幕阅读器友好
- 焦点指示清晰

## Implementation Notes

**技术栈（复用 v3.0）：**
- Frontend: React + TypeScript
- UI Library: Tailwind CSS + shadcn/ui
- Charts: Recharts
- State: Zustand

**新增组件：**
- `<EvolutionDemo />` - 首次进化 Demo
- `<EvolutionProgress />` - 进化进度
- `<ConflictDetector />` - 冲突检测
- `<StatsHeatmap />` - 统计热力图
- `<EffectivenessTrend />` - 有效性趋势

**性能要求：**
- 首屏加载 <2 秒
- 图表渲染 <500ms
- 交互响应 <100ms
